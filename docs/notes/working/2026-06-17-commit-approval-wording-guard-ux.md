# commit approval wording guard UX

作成日: 2026-06-17

## 背景

commit 実行代行の guard で、人間には意図が分かる承認文言でも、機械的には
「commit 実行代行を明示的に許可する文」として不足する場合がある。

例:

- 「コミットを分けて処理」は、分割方針や stage の承認としては読める。
- しかし guard にとっては、commit 実行そのものを代行してよいという明示承認とは限らない。

## 改善したい点

guard を緩めるのではなく、拒否時の説明と次アクションを明確にする。

- guard のエラーに、必要な承認文の例を出す。
- stage 承認、commit 実行承認、push 承認を別概念として表示する。
- 不足している承認要素を列挙する。
- commit 操作カードに、必要な文言と承認単位を短く固定する。

## 方針案

安全側の拒否は維持する。

そのうえで、利用者が次に何を言えば進められるかを、guard または操作カードが
その場で示せるようにする。
