    description: >

    rationale: >
      If the structured summary omits or softens material upstream constraints from requirements.md,

      The review would silently operate on an incomplete or biased representation of upstream intent.
  - severity: WARN
    target_location: "Proposed Criteria / Required Checks / Check 5 and Out Of Scope"
    description: >
      Two separate sections repeat the instruction not to treat mere presence of workflow-operation
      code as a violation, with no countervailing instruction to actively probe for subtle gate
      bypasses. The combined leniency bias may cause the reviewer to underreport CRITICAL
      boundary-weakening behavior.
    rationale: >
      The most important failure mode for an approval-gate review is a subtle code path that
      allows proxy_model to satisfy a human_only boundary. The current framing tips the reviewer
      toward leniency rather than active detection, which is the opposite of the required posture
      for CRITICAL-severity findings.

    target_location: "Proposed Criteria / Review Target / target manifest note"
    description: >
      The criteria instructs the reviewer to report ERROR when a target path is absent from the
      injected prompt, but an absent target makes the entire review invalid and should be CRITICAL.
    rationale: >
      An ERROR implies the review can continue with reduced confidence. A missing target means the
      reviewer has no artifact to judge and must not return any implementation findings. CRITICAL
      is the appropriate severity to halt the review run.

      Requirements.
    rationale: >
      Prohibited actions need a severity anchor in Finding Policy so the reviewing model has an
      unambiguous instruction to escalate, not merely exclude, any output that crosses those
      boundaries.
