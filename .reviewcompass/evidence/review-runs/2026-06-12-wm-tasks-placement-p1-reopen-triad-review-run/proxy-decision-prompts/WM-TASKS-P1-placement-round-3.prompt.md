# proxy_model 判断依頼：wm tasks triad-review round-3 の三段階トリアージ

あなたは ReviewCompass の proxy_model（人間判断の代行役）である。workflow-management tasks の round-3
（round-1 X1・round-2 Y1/Y2 適用後の収束確認）では、主役（claude）所見ゼロ、敵対役（gpt-5.5）と判定役（gemini）が
同根の 2 クラスタを指摘した。4 所見の最終ラベルを判断せよ。不可逆操作の判断は範囲外。

## 文脈

- 現行 T-004 の凍結期挙動テストは「3 パス × 3 観点」（(1) 新配置への書き込み (2) 旧配置への新規書き込みなし
  (3) 旧パスにしか記録がない場合の新→旧フォールバック読み取り）＋round-2 の境界条件（効力発生時点・暗黙無効化なし）。
- design の確定契約（§実行時生成物の凍結期の扱い）は「読み取りは新→旧の順」「既存分は旧置き場で凍結
  （移動・上書き・追記をしない）」を正本化済み。
- 先行の ce reopen では同型の契約に対し「新旧同名時の新配置優先＋警告」テストと「git 追跡履歴による凍結違反
  （追加・変更・削除）検出」テストを tasks 要件化し実装済み（先行事例）。

## 同根クラスタと候補案

### T1：新旧競合時の「新配置優先」を機械検証する要件が欠落（gpt-5.5-adversarial-001〔ERROR〕・gemini-3.1-pro-preview-judgment-001〔ERROR〕、同根）
- 指摘の要点：観点 3 は「旧のみ存在」のフォールバック確認に留まり、新旧両方に記録がある競合ケースで新配置が
  採用されることを検出できない。実装が旧→新の順でも現行テスト要件を通過し得る。
- 候補案：
  - A：凍結期挙動テストへ観点 4「新旧競合時の新配置優先」を追加（3 パスそれぞれで、新旧に異なる内容を置き、
    新配置の内容が採用されることを検証）。完了条件 5 も 4 観点へ拡張（操縦 LLM の推薦案。ce の同型テストと整合）
  - B：現状維持（観点 3 で足りるとみなす）
- 利点と弱点：A は design 契約「新→旧の順」を直接 TDD で固定する。B は ERROR 2 件の指摘どおり検出漏れが残る。

### T2：凍結済み旧成果物の不変性検証の粒度が未明示（gpt-5.5-adversarial-002〔WARN〕・gemini-3.1-pro-preview-judgment-002〔WARN〕、同根）
- 指摘の要点：「旧配置への新規書き込みなし」だけでは、既存旧ファイルの削除・移動・内容変更が検証されない。
- 候補案：
  - A：凍結期挙動テストへ観点 5「凍結済み旧成果物の不変性」を追加（P1 実装反映コミット以降に旧 3 パスの既存
    ファイルが変更・削除されていないことを git 追跡履歴で検出。ce の凍結違反検出と同じ判定規則）。完了条件 5 も追従
    （操縦 LLM の推薦案）
  - B：design の共通契約（移動・上書き・追記をしない）に委ね、tasks の機械検証は追加しない
- 利点と弱点：A は監査証跡の保全を機械検証化し ce と対称になる。B は契約が文書上の約束に留まる
  （ce では機械検証化を採用済み）。
