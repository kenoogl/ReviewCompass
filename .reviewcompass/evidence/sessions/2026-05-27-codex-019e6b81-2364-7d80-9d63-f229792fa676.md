## 仕組み

macOS の **LaunchAgent**（= ログイン時に自動実行される小さな仕掛け）を使い、毎回ログインのたびに `launchctl setenv` を自動実行させます。これにより、デスクトップアプリも環境変数を見られる状態になります。

設定ファイルは `~/Library/LaunchAgents/` ディレクトリの中に **`.plist`**（= Apple 形式の設定ファイル）として置きます。

## 手順（OPENAI_API_KEY の例）

**手順 1：設定ファイルを作る**

`~/Library/LaunchAgents/setenv.openai.plist` というファイル名で、次の内容を保存：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTD/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>setenv.openai</string>
  <key>ProgramArguments</key>
  <array>
    <string>launchctl</string>
    <string>setenv</string>
    <string>OPENAI_API_KEY</string>
    <string>ここに実際のキー値</string>
  </array>
  <key>RunAtLoad</key>
  <true/>
</dict>
</plist>
```

`ANTHROPIC_API_KEY` 用は同様に `setenv.anthropic.plist` を別に作ります。

**手順 2：権限を厳しくする（重要）**

```
chmod 600 ~/Library/LaunchAgents/setenv.openai.plist
chmod 600 ~/Library/LaunchAgents/setenv.anthropic.plist
```

これで利用者本人だけが読めるようになります。

**手順 3：読み込ませる**

```
launchctl load ~/Library/LaunchAgents/setenv.openai.plist
launchctl load ~/Library/LaunchAgents/setenv.anthropic.plist
```

**手順 4：デスクトップアプリを `Cmd + Q` で終了 → 再起動**

## 重大な注意点

1. **キーが平文（暗号化されない生の文字列）でファイルに残ります**：誰かが `~/Library/LaunchAgents/` を読める状態なら、キーが盗まれます。手順 2 の `chmod 600` を **必ず**実行してください
2. **Time Machine などのバックアップにキーが含まれます**：バックアップ先の管理にもご注意を
3. **macOS のバージョンによって `launchctl` の作法が異なる場合があります**：最近の macOS（Sonoma 以降）では `launchctl bootstrap gui/$(id -u) <ファイル>` 形式が推奨されることがあります。手順 3 でエラーが出たらお知らせください
4. **キーを変更したいとき**：ファイルを書き換えて `launchctl unload` → `load` を再実行する手間がかかります
