## 何が起きたか

エラーは `~/.zshrc` の5行目、PATH（実行ファイルを探すディレクトリ一覧）を組み立てる部分です：

```zsh
path=(
  /opt/homebrew/bin(N-/)
  ...
)
```

各ディレクトリの末尾に付いている **`(N-/)`** が引っかかりました。

## `(N-/)` の意味

これは zsh の「グロブ修飾子」（glob qualifier ＝ ファイル名展開のときに、対象を後ろから絞り込む指定）です。3つの文字それぞれに意味があります：

- **`N`** … 該当ディレクトリが存在しなくてもエラーにせず、その項目を消す（存在しない場所を PATH から黙って除外するため）
- **`-`** … シンボリックリンク（ある場所への別名・近道）の実体をたどって判定する
- **`/`** … ディレクトリだけを対象にする

つまり「`/opt/homebrew/bin` が実在するディレクトリなら PATH に入れ、無ければ黙って飛ばす」という、安全に PATH を組む書き方です。**.zshrc 自体は正しく、壊れていません。**

## なぜエラーになったのか（再現で確認）

この記法が有効かどうかは、zsh の **`BARE_GLOB_QUAL`** というオプション（末尾の `(...)` を修飾子として解釈する設定）が入っているかで決まります。

- 普通の対話シェル（あなたが端末で使う zsh）では、このオプションは**初期値でオン**。だから .zshrc は問題なく読み込まれます。
- ところが、私がツールから `source ~/.zshrc` を実行したときの内部シェルは、このオプションが**オフ**の状態でした。オフだと zsh は `(N-/)` を修飾子と認識できず、普通のファイル名パターンとして解釈しようとして失敗し、「bad pattern」と言います。

実際に切り分けて再現しました：

| 実行 | 結果 |
|------|------|
| `zsh -c 'path=(/opt/homebrew/bin(N-/))'`（既定オン） | 正常（`ok: /opt/homebrew/bin`） |
| `unsetopt bareglobqual` を入れてから同じ実行 | `bad pattern: /opt/homebrew/bin(N-/)`（今回と同一） |

このオプションがオフになるのは、zsh が `sh` 互換モード（POSIX 互換の動作）で起動された場合などです。ツールがコマンドを実行する内部の起動方法が、その状態だったためです。

- **あなたの環境は何も直す必要はありません。** 端末で普通に使う分には .zshrc は正しく動きます。
